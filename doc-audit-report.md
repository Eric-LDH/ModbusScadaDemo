### B. Modbus 寄存器映射文档模板

```markdown
# Modbus RTU 寄存器映射表

> 适用版本：v1.0 | 从站地址：1 | 通信参数：9600-8-N-1

## 通信参数

| 参数 | 值 | 备注 |
|------|-----|------|
| 物理层 | RS-485 / RS-232 | 取决于串口适配器 |
| 波特率 | 9600 bps | 硬编码，与从站设备匹配 |
| 数据位 | 8 | — |
| 校验位 | None | — |
| 停止位 | 1 | — |
| 从站地址 | 1 | 常量 `SlaveAddress` |
| 轮询周期 | 1000 ms | 1 秒间隔 |
| 超时 | 读/写各 1000 ms | SerialPort.ReadTimeout/WriteTimeout |

## 保持寄存器（功能码 03 - Read Holding Registers）

| 地址 | 偏移 | 名称 | 数据类型 | 单位/倍率 | 范围 | 读写 | 描述 |
|------|------|------|---------|----------|------|------|------|
| 40001 | 0x00 | 温度值 | `uint16` | ℃, ÷100 | 0~65535 (0~655.35℃) | R | 从站当前温度，存储值=实际值×100。例：寄存器值=2550 → 25.50℃ |
| 40002 | 0x01 | 继电器状态 | `uint16` | — | 0 或 1 | R | 0=继电器断开(风扇OFF)，1=继电器闭合(风扇ON) |

> **读取方式**：`master.ReadHoldingRegisters(1, 0, 2)` — 从地址0开始连续读2个寄存器

## 线圈（功能码 05 - Write Single Coil）

| 地址 | 偏移 | 名称 | 数据类型 | 有效值 | 读写 | 描述 |
|------|------|------|---------|--------|------|------|
| 00001 | 0x00 | 风扇控制 | `bool` | TRUE/FALSE | W | TRUE=开风扇(继电器吸合)，FALSE=关风扇(继电器断开) |

> **写入方式**：`master.WriteSingleCoil(1, 0, true/false)`

## 寄存器布局图

```
保持寄存器:
  0x00 ┌──────────────┐
       │   温度值×100  │  ReadHoldingRegisters(1, 0, 2)
  0x01 ├──────────────┤
       │  继电器状态   │
       └──────────────┘

线圈:
  0x00 ┌──────────────┐
       │   风扇控制     │  WriteSingleCoil(1, 0, value)
       └──────────────┘
```

## 轮询时序

```
时间轴:  T0      T+1s    T+2s    T+3s    T+4s    T+5s
         │ Read  │       │ Read  │       │ Read+Save │
         │ HR[2] │       │ HR[2] │       │ HR[2]→DB  │
         ▼       ▼       ▼       ▼       ▼           ▼
轮询:   [■]     [■]     [■]     [■]     [■]  ← 1秒间隔
DB存储:                                         ■   ← 每5次(5秒)
```

## 错误处理

| 错误场景 | Modbus 响应/异常 | 应用行为 |
|---------|-----------------|---------|
| 从站无响应 | TimeoutException | 记录日志 → 断开连接 → 启动自动重连（如启用） |
| CRC 校验错误 | Modbus 协议层异常 | 同上 |
| 串口被占用 | IOException | 弹窗提示用户 |
| 从站返回异常码 | SlaveException | 记录日志 → 同上 |
```

## 3. 启动检查清单

### 3.1 环境检查

- [ ] **.NET Framework 4.7.1 已安装**
  - 检查方法：控制面板 → 程序和功能 → 已安装的更新
  - 如未安装：https://dotnet.microsoft.com/download/dotnet-framework/net471

- [ ] **串口设备已连接且未被占用**
  - 检查方法：设备管理器 → 端口(COM 和 LPT)，确认设备存在
  - 检查占用：关闭其他可能使用该 COM 口的软件（串口调试助手等）

- [ ] **Modbus 从站设备已上电**
  - 确认从站设备电源指示灯正常
  - 确认 RS-485 接线：A+/B- 正确连接

- [ ] **程序目录可写入**
  - 确认 `scada.log` 和 `scada_data.db` 所在目录有写入权限

### 3.2 启动步骤

1. 双击运行 `ModbusScada.exe`（或执行 `build_and_run.bat` 自动编译并启动）
2. 在「串口设置」面板中，下拉选择对应 COM 口（若列表为空，点击「刷新」）
3. 点击「连接」按钮
4. 等待 1~2 秒，观察状态栏：显示「● 已连接 | COMx」即为成功
5. 温度值应当开始更新，曲线开始绘制

### 3.3 启动后验证

- [ ] 状态栏显示绿色「● 已连接」
- [ ] 温度数值正常刷新（非「通信异常」）
- [ ] 实时曲线正常滚动
- [ ] 继电器状态标签有颜色（绿色=ON/红色=OFF）

## 4. 常见故障排查

### 4.1 串口列表为空

| 原因 | 排查步骤 |
|------|---------|
| 设备未连接/驱动未安装 | 检查设备管理器 → 端口，确认 COM 口存在 |
| 权限不足 | 以管理员身份运行程序试试 |

### 4.2 点击连接后提示"连接失败"

| 原因 | 排查步骤 |
|------|---------|
| COM 口被其他软件占用 | 关闭串口调试工具、其他 SCADA 软件 |
| 串口参数不匹配 | 确认从站设备波特率=9600, 数据位=8, 校验位=None, 停止位=1 |
| USB 转串口驱动异常 | 重新插拔 USB 转串口适配器，或重装驱动（CH340/CP2102/FT232） |

### 4.3 温度显示"通信异常"

| 原因 | 排查步骤 |
|------|---------|
| 从站设备断电/故障 | 检查从站电源，验证 Modbus 通信（可用 Modbus Poll 工具测试） |
| RS-485 线缆松动 | 检查 A+/B- 接线端子 |
| 从站地址不匹配 | 确认从站地址=1（或用 Modbus Poll 扫描地址） |
| 串口通信中断 | 查看 `scada.log` 中 ERROR 日志 |

### 4.4 温度曲线不更新

| 原因 | 排查步骤 |
|------|---------|
| 通信已断开 | 检查状态栏连接状态 |
| UI 线程阻塞 | 重启程序 |
| ScottPlot 渲染异常 | 查看 `scada.log` 是否有相关异常 |

### 4.5 开/关风扇无响应

| 原因 | 排查步骤 |
|------|---------|
| 未连接 | 确认已连接状态 |
| Modbus 写线圈超时 | 查看 `scada.log` 中的错误日志 |
| 继电器硬件故障 | 用万用表测量继电器输出端 |

### 5.1 文件位置

| 文件 | 默认路径 | 用途 |
|------|---------|------|

### 5.2 日志格式

```
2026-05-27 10:15:30.1234 | INFO | 程序启动
2026-05-27 10:15:31.5678 | INFO | 串口已连接: COM3 9600-8-N-1
2026-05-27 10:20:45.9012 | WARN | 温度报警触发! 当前温度: 52.3℃, 阈值: 50℃
2026-05-27 10:22:00.3456 | ERROR | Modbus 轮询读取失败
```

### 5.3 数据库表结构

```sql
-- temperature_log 表
CREATE TABLE temperature_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp   DATETIME DEFAULT CURRENT_TIMESTAMP,
    temperature REAL,        -- 温度值 (℃)
    relay_state INTEGER,     -- 继电器状态 (0=OFF, 1=ON)
    alarm_state INTEGER      -- 报警状态 (0=正常, 1=报警)
);

-- 查询最近24小时数据
SELECT * FROM temperature_log
WHERE timestamp >= datetime('now', '-1 day')
ORDER BY timestamp DESC;

-- 导出为 CSV
-- 可使用 DB Browser for SQLite 工具，或执行：
-- .mode csv
-- .headers on
-- .output export.csv
-- SELECT * FROM temperature_log;
```

## 2. 操作步骤

### 2.1 启动和连接

1. 双击 `ModbusScada.exe` 启动程序
2. 在左侧「串口设置」面板，确认 COM 口已正确选择（若为空，点击「刷新」）
3. 点击「**连接**」按钮
4. 观察：
   - 连接按钮变为红色「断开」
   - 状态栏显示绿色「● 已连接 | COMx」
   - 温度值开始更新
   - 曲线开始绘制

### 2.2 设置报警阈值

1. 在「报警设置」面板，点击报警阈值数值框
2. 输入目标温度值（如 45.0），或点击上下箭头微调
3. 曲线上的红色虚线会自动更新到新阈值位置
4. 当温度连续 3 次超过阈值时，触发报警（红色警告）

### 2.3 手动控制风扇

1. 确保已连接状态（「开风扇」「关风扇」按钮可用）
2. 点击「**开风扇**」→ 继电器吸合，风扇启动
3. 点击「**关风扇**」→ 继电器断开，风扇停止
4. 中间面板的「继电器状态」标签会实时反馈状态

### 2.4 读取历史数据

历史数据存储在 `scada_data.db`（SQLite 数据库）中，建议使用以下工具查看：

- **DB Browser for SQLite**（推荐，免费）：https://sqlitebrowser.org/
- 或使用命令行：`sqlite3 scada_data.db "SELECT * FROM temperature_log LIMIT 10"`

### 2.5 断开连接

- 点击「**断开**」按钮
- 或直接关闭程序窗口（自动断开并清理资源）

## 3. 报警机制说明

| 特性 | 说明 |
|------|------|
| 触发条件 | 温度连续 **3 次** 超过设定阈值 |
| 解除条件 | 温度连续 **3 次** 低于设定阈值 |
| 判断周期 | 每次轮询（1 秒）判断一次 |
| UI 表现 | 报警状态标签变红显示"⚠ 温度过高!"，温度背景变浅红 |
| 日志记录 | NLog 写入 WARN 级别日志 |

> 死区防抖设计避免了温度在阈值附近波动导致的频繁报警/解除。

## 4. 通信状态说明

| 状态指示 | 含义 |
|---------|------|
| ● 已连接 (绿) | Modbus 通信正常 |
| ● 未连接 (灰) | 未建立连接 |
| 温度显示「通信异常」 | 已连接但数据读取失败（可能是从站断电或线缆断开） |
| 自动重连提示 | 如勾选了自动重连，通信故障后程序会每 3 秒尝试重连 |

## 5. 常见问题

| 问题 | 解决方法 |
|------|---------|
| Q: COM 口列表为空 | A: 点击「刷新」，检查设备管理器确认串口驱动正常 |
| Q: 连接失败 | A: 确认 COM 口未被其他软件占用，确认从站设备已上电 |
| Q: 温度持续显示"通信异常" | A: 检查 RS-485 接线，确认从站地址=1，尝试重新连接 |
| Q: 曲线不显示 | A: 确保已连接且温度数据正常读取，尝试调整窗口大小 |
```

### 1. 代码注释 → API 文档：自动生成

| 工具 | 适用性 | 推荐度 |
|------|--------|--------|
| **DocFX** | 支持 C# XML 注释 → 静态 HTML 站点；可集成 Markdown 补充文档 | ⭐⭐⭐⭐⭐ 首推 |
| **Sandcastle Help File Builder** | 传统 MSDN 风格 CHM/HTML 文档 | ⭐⭐⭐ 备选 |
| **Doxygen** | 跨语言，但对 .NET XML 注释支持不如 DocFX | ⭐⭐ |

**推荐配置**：
```yaml
# docfx.json
{
  "metadata": [
    {
      "src": [{ "files": ["ModbusScada.csproj"] }],
      "dest": "api"
    }
  ],
  "build": {
    "content": [
      { "files": ["api/*.yml"] },
      { "files": ["docs/**/*.md"] }
    ],
    "dest": "_site"
  }
}
```

**CI 集成**：在构建脚本中添加 `docfx build` 并发布 `_site` 到内网文档服务器。

| 文档拼写检查 | `cspell` + 术语词典 | PR/MR 时 |

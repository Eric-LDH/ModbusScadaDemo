### 界面分区说明

| 区域 | 位置 | 功能 |
|------|------|------|
| **串口设置** | 左上 | COM 口选择、连接/断开、自动重连开关 |
| **报警设置** | 左下 | 温度报警阈值设置、报警状态显示 |
| **温度与控制** | 中间 | 实时温度大字显示、继电器状态、风扇按钮 |

| **状态栏** | 底部 | 连接状态 + 报警状态 |

## 2. 操作流程

### 2.1 启动与连接

**步骤 1：扫描串口**

1. 确保下位机（Modbus 从站设备）已通过串口线连接到电脑
2. 点击"串口设置"区域的 **[刷新]** 按钮
3. 下拉列表将显示系统中可用的 COM 口

**步骤 2：选择并连接**

1. 在下拉列表中选择目标串口（如 COM3）
2. 点击 **[连接]** 按钮
3. 连接成功后：
   - 按钮变为红色的"断开"
   - 状态显示为绿色的"● 已连接"
   - 温度数值开始更新
   - 曲线开始绘制

**步骤 3：启用自动重连（可选）**

- 勾选"□ 自动重连"复选框
- 当通信意外中断时，系统将每 3 秒自动尝试重连

### 2.2 查看温度

连接成功后，"温度与控制"区域将实时显示：

| 显示状态 | 含义 |
|----------|------|
| `25.5` | 正常温度值（℃），深色字体 |
| `25.5 ⚠传感器异常` | 传感器可能故障，橙色警告 |
| `通信异常` | 与下位机通信中断，灰色字体 |

右侧曲线图同步更新温度趋势，Y 轴自动缩放以适应数据范围。

### 2.3 设置报警阈值

1. 在"报警设置"区域，点击数字框
2. 输入或通过上下箭头调整阈值（范围 0~200℃，精度 0.1℃）
3. 阈值修改即时生效

报警状态指示：

| 显示 | 颜色 | 含义 |
|------|------|------|
| `✓ 温度正常` | 绿色 | 温度在安全范围内 |
| `⚠ 温度过高!` | 红色 | 温度超过阈值（连续 3 次确认） |

> **死区说明**：系统采用防抖机制。温度需要**连续 3 次**超过阈值才触发报警，同样需要**连续 3 次**低于阈值才解除。这避免了阈值附近的频繁抖动报警。

### 2.4 手动控制风扇

> ⚠️ **注意**：手动控制需要下位机已连接且 Modbus 通信正常。

**开风扇**：

1. 点击 **[开风扇]** 按钮
2. 在确认弹窗中点击"是"
3. 系统将向从站写入线圈 0 = ON

**关风扇**：

1. 点击 **[关风扇]** 按钮
2. 在确认弹窗中点击"是"
3. 系统将向从站写入线圈 0 = OFF

每次手动操作都会记录审计日志。

### 2.5 断开连接

- 点击红色的 **[断开]** 按钮即可断开串口连接
- 关闭程序窗口将自动断开连接并保存数据

## 3. 故障排查

### 3.1 连接失败

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 点击连接后弹错误框 | 串口被占用 | 关闭其他串口软件，重试 |
| 连接成功但无数据显示 | 从站地址不匹配 | 确认从站地址为 1，或修改代码中的 `SlaveAddress` |
| 连接成功但温度显示"通信异常" | 通信超时 | 检查物理连接和从站供电 |
| 串口列表为空 | 驱动未安装 | 安装 USB 转串口驱动，检查设备管理器 |
| 自动重连反复失败 | 串口硬件问题 | 检查物理连接，确认从站正常工作 |

### 3.2 数据显示异常

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 温度显示为 0 且无变化 | 从站返回数据为 0 | 用 Modbus Poll 验证从站数据 |
| 温度跳变严重 | 传感器故障或干扰 | 检查传感器接线和屏蔽 |
| 显示"⚠传感器异常" | 数据超范围或跳变过大 | 检查传感器是否损坏 |
| 继电器状态不更新 | 寄存器映射错误 | 确认从站寄存器 1 为继电器状态 |

### 3.3 报警异常

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 温度超阈值但不报警 | 死区需要连续 3 次确认 | 等待 3 秒以上 |
| 温度低于阈值但报警不解除 | 死区需要连续 3 次低于阈值 | 等待 3 秒以上 |
| 报警阈值无法修改 | 界面控件锁定 | 检查是否在非连接状态下操作 |

## 4. 日志文件

程序运行时会自动生成日志文件，位于程序所在目录：

| 文件 | 说明 |
|------|------|
| `scada.log` | 当日日志 |
| `scada.20260715.log` | 历史归档（按日期命名） |

如需排查问题，可将日志文件提供给技术支持人员。

---

## 5. 数据库文件

温度历史数据存储在 `scada_data.db`（SQLite 格式），位于程序所在目录。

可使用以下工具查看历史数据：
- **DB Browser for SQLite**（免费）
- **Navicat for SQLite**
- 命令行：`sqlite3 scada_data.db "SELECT * FROM temperature_log ORDER BY id DESC LIMIT 100;"`
